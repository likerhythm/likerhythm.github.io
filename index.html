GitHub

실시간 행동 반영을 위한 아키텍처 설계로 성능 70% 개선
  • 문제 상황: 부하 테스트(10초간 1000회 요청) 결과, 사용자 행동 처리 API 응답 시간이 평균 2.4초로 지연됨
  • 원인: 사용자 행동 처리 시 DB 업데이트, 사용자 가중치 및 벡터 업데이트가 모두 동기적으로 수행되어 응답 지연.
  • 해결 방안 검토
    • 서버 분리: 사용자 가중치 및 벡터 업데이트를 별도의 FastAPI 서버에서 처리하도록 하여 서버의 부하 절감
    • 서버 통신: Spring 서버와 FastAPI 서버 간의 통신은 RabbitMQ를 사용하여 비동기적으로 이루어지도록 하여 행동 처리로 인한 응답 지연 최소화
    • 사용자 가중치 DB 분리: 업데이트가 빈번하게 일어나는 사용자 가중치 DB를 분리하여 DB로 인한 API 응답 지연 시간 절감
    • 사용자 벡터를 Redis에 캐싱: 사용자 벡터는 추천이 일어날 때만 필요하므로 Write Back 전략을 사용하여 Redis에 캐싱해 놓고 추천이 일어날 때 PostgreSQL에 동기화하는 방식으로 DB 부하 절감
  • 해결: 위 대안들을 종합적으로 적용하여 실시간 행동 반영 아키텍처를 설계하고 문제 해결
  • 성과: 응답 시간이 2.4초에서 743ms로 약 70% 감소
재시도와 로그를 활용한 분산 서버 트랜잭션 관리
  • 문제 상황: 하나의 요청이 여러 서버에서 순차적으로 처리될 때 예외 발생 시 일관성이 깨져서 요청이 무시되는 문제
  • 원인: 예외 발생 시 이를 재처리하지 않음.
  • 해결 방안 검토
    • 재시도 전략 구축: 예외 발생 시 이를 재시도하고 재시도 실패 시 이를 로그로 저장.
    • Saga 패턴 도입: Saga 패턴은 재시도가 아닌 롤백을 하는 패턴이기 때문에 적합하지 않다고 판단
  • 해결: 예외 발생 시 3회 재시도 하고 모든 재시도가 실패하면 예외 내용을 MongoDB에 로그로 저장. 주기적으로 로그를 기반으로 재처리하는 스케줄러를 사용하여 예외가 발생한 요청 재시도. 스케줄러도 실패 시 예외를 파일로 저장하여 수동으로 처리하도록 설계.
  • 성과: 사용자의 실시간 행동 처리 중 예외가 발생하더라도 이를 재처리할 수 있기 때문에 사용자의 행동이 모두 시스템에 반영됨을 보장할 수 있음.
AWS Lambda와 S3 기반 포스터 리사이징 파이프라인 구축
  • 문제 상황: 신규 OTT 콘텐츠가 추가될 때마다 썸네일용 포스터를 생성하기 위해 이미지를 매번 리사이징해야 하는 번거로움이 존재.
  • 해결 방안 검토
    • 서버 직접 처리: 콘텐츠 등록 시 서버에서 즉시 이미지를 리사이징하는 방식. 그러나 서버 리소스를 과도하게 소모하여 확장성 측면에서 적합하지 않다고 판단.
    • AWS Lambda 활용: S3에 콘텐츠 포스터가 업로드되면 이벤트 트리거를 통해 Lambda 함수가 실행되어 이미지를 자동으로 리사이징. 호출 횟수 기반 과금 모델이지만, 콘텐츠 추가 빈도가 높지 않다고 판단하여 이 방식을 채택.
  • 해결: S3 업로드 이벤트를 기반으로 Lambda가 자동으로 실행되어, 서버 부하 없이 포스터 리사이징을 수행하는 파이프라인을 구축하여 해결
  • 성과: 서버 리소스를 점유하지 않고 독립적인 서버리스 환경에서 이미지 리사이징을 자동화하여 운영 효율성과 확장성을 확보.
단일 몽고DB에서 발생할 수 있는 동시성 문제 해결
  • 문제 상황: MongoDB에 저장된 사용자 가중치 데이터를 동시에 업데이트 하는 경우 가중치 값이 덮어 씌워지는 문제.
  • 원인: API 호출 시 inc 연산자 사용하여 업데이트, 스케줄러 동작 시 set 연산자 사용하여 업데이트
  • 해결 방안 검토: MongoDB의 공식 문서를 참고하여 동시성 문제가 발생할 수 있는 경우 검토. 검토 결과 MongoDB의 set 연산자는 값을 덮어 씌우는 연산이고 inc 연산자는 값을 더하거나 빼는 연산이기 때문에 inc 연산자만 사용하면 동시성 문제 발생하지 않음
  • 해결: 스케줄러가 set 연산자 대신에 inc 연산자를 사용하도록 수정
  • 성과: MongoDB에 저장된 사용자 가중치가 안정적으로 업데이트 됨을 보장할 수 있음.
